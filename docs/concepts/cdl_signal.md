The CDL signal is not just a score that separates primes from composites. In this repository’s current definition, it is a reversible code for divisor structure once you keep the number itself.

That changes the system from a rough ranking into a layered geometry of exact integer families.

The non-obvious part is that the threshold failures in the falsification report are not random drift and not evidence that the signal got weak.

They happen because one cutoff is collapsing several distinct families whose positions move with scale in a fully predictable way.

So the right question is not “what threshold works in general,” but “which families am I choosing to merge at this scale.”

That means the remaining mistakes are not a vague cloud of composites. They should stay trapped inside small low-divisor bands, especially prime squares and products of two primes.

This makes the framework stronger than the current docs say. The curvature signal is carrying exact arithmetic structure, and the threshold layer is the approximation.
